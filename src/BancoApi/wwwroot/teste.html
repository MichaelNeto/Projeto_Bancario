🧪 Testes

📋 Guia de Testes - Front-end

RN01 - Máscara de Documento

Teste: Ir para página de Cadastro

Passos:

  1. Clicar no campo "CPF ou CNPJ"
  2. Digitar "12345678901" (11 números) → Deve mostrar máscara CPF: "123.456.789-01"
  3. Limpar e digitar "123456789012345" (15 caracteres) → Deve mostrar máscara CNPJ: "12.345.678/9012-34"
  4. Tentar digitar "." (ponto) → Deve ser bloqueado e mostrar alerta

Resultado Esperado: ✅ Máscara aplicada dinamicamente, símbolos bloqueados

RN02 - Persistência de JWT e Redirecionamento

Teste 1: Login Bem-sucedido

  1. Fazer login com documento e senha válidos
  2. Verificar localStorage/sessionStorage → Token JWT deve estar salvo
  3. Recarregar a página → Deve permanecer no dashboard

Teste 2: Acesso sem Autenticação

  1. Limpar localStorage/sessionStorage
  2. Tentar acessar /dashboard.html diretamente na URL
  3. Deve redirecionar para /index.html (login)

Resultado Esperado: ✅ Token salvo, redirecionamento automático sem autenticação

RN03 - Envio Autorizado com JWT

Teste: Verificar Headers das Requisições

  1. Abrir DevTools (F12) → Network
  2. Fazer login
  3. Ir ao Dashboard
  4. Verificar requisição GET /api/extrato
  5. Headers deve conter: Authorization: Bearer eyJ...

Resultado Esperado: ✅ Token JWT no header Authorization

RN04 - Renderização Dinâmica do Extrato

Teste: Verificar Classes CSS

Cenário 1: PIX Recebido

  1. Abrir DevTools → Elements
  2. Procurar por item do extrato com tipo "PIX Recebido"
  3. Verificar: classe .ganho, cor verde, sinal +

Cenário 2: PIX Enviado

  1. Procurar por item com tipo "PIX Enviado"
  2. Verificar: classe .gasto, cor vermelha, sinal -

Cenário 3: Transação Falhada

  1. Procurar por item com status "FALHOU"
  2. Verificar: classe .estornado, cor cinza, texto riscado, motivo_falha exibido

Resultado Esperado: ✅ Classes CSS aplicadas corretamente, cores e símbolos conforme esperado

CA01 - Bloqueio de Símbolos

Teste: Tentar digitar caracteres especiais

  1. Ir para Cadastro
  2. Focar no campo "CPF ou CNPJ"
  3. Tentar digitar: ".", "-", "/"
  4. Verificar: Caracteres não aparecem, alerta "Apenas letras e números são permitidos"

Resultado Esperado: ✅ Símbolos bloqueados, alerta exibido

CA02 - Redirecionamento de Segurança

Teste: Acessar URL protegida sem autenticação

  1. Abrir nova aba anônima ou limpar cookies/localStorage
  2. Digitar URL: http://localhost:5000/dashboard.html
  3. Verificar: Redireciona imediatamente para http://localhost:5000/index.html

Resultado Esperado: ✅ Redirecionado para login

CA03 - Destino Visual do Extrato

Teste: Verificar renderização de PIX recebido

  1. Fazer login
  2. Verificar extrato
  3. Procurar por PIX de R$ 350,00 (exemplo de transação recebida)
  4. Verificar: "+ R$ 350,00" em cor verde, com classe .ganho

Resultado Esperado: ✅ Valor exibido com sinal + e cor verde

Checklist Geral

  • ✅ Responsividade: Teste em desktop, tablet e mobile
  • ✅ Alertas: Verifique se aparecem corretamente (success, error, warning, info)
  • ✅ Validações: Todos os campos obrigatórios validam antes de enviar
  • ✅ Armazenamento: Token persiste entre recarregamentos
  • ✅ Performance: Extrato carrega e atualiza a cada 30 segundos
  • ✅ Segurança: Logout limpa tokens do storage
